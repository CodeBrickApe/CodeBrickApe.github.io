ipad使用Vue&Element的产生的Bug

发布于 2021年08月23日100 PV

主要记录一下，公司已经在使用的pc后台管理兼容ipad端使用，开发技术Vue、Element，主要是以下几个问题：

  • 金额输入框金额校验、ipad弹出键盘可输入任意字符（input的type=Number无效）
  • 下拉框el-select在ipad上点击下拉框区域展开与隐藏无效
  • ipad点击事件第一次无效，必须点击两次

一、金额输入框金额校验、ipad弹出键盘可输入任意字符（input的type=Number无效）

将input的type类型设置为Number，在PC端&手机端输入时仅能输入数字类型，但是在iPad上单击输入框弹出的是全键盘，可以切换中英文输入包括标点符号，而切触发input的onBlur和onChange事件时，获取不到value值。在网上看了许多解决方案，基本有以下几种：

  1. 使用原生input，设置属性（pattern="[0-9]*"）| 测试后发现在ipad上没有效果
<input type="number" pattern="[0-9]*"  placeholder="请输入..." />
  1. 在input框的value值加正则校验 通过正则能保证不输入非数字，但是用户打开的是26键键盘，需要再切换到数字键盘，没有采用此方法
<input type="text" id="class_rate_percent" class="wsinput" value="" onkeyup="value=value.replace(/[^\d.]/g,'')"/>

最后发现将input的type设置为tel，既能保证用户只能输入数字还能输入小数，而且通过正则完成对输入金额的校验，具体实现代码在下面展示：

html

<el-input  
	suffix-icon="el-icon-edit"
	type='tel'
	v-model="contractInfo.money"
	@input="numberChange"  //在 Input 值改变时触发
	@change="numberChange" // 仅在输入框失去焦点或用户按下回车时触发
> </el-input>

javascript

// input框发生改变，检查类型&值的范围
    numberChange (event,value) {
      // var keyValue = event ? event.target.value : value
      var keyValue = event
      console.log(keyValue);
      //先把非数字的都替换掉，除了数字和.
      keyValue =keyValue.replace(/[^\d.]/g,"");
      //保证只有出现一个.而没有多个.
      keyValue = keyValue.replace(/\0.{2,}/g,".");
      //保证.只出现一次，而不能出现两次以上
      keyValue =keyValue.replace(".","$#$").replace(/\./g,"").replace("$#$",".");
      //保证.后只出现2位
      keyValue =keyValue.replace(/([0-9]+\.[0-9]{2})[0-9]*/,"$1");
      let integer = /^[+]{0,1}(\d+)$|^[+]{0,1}(\d+\.\d+)$/;  
      if (integer.test(keyValue) && keyValue > this.maximum) {
          this.$message({
            showClose: true,
            message: '输入金额不可以大于应付金额！',
            type: 'warning'
          });
          let newVal =Math.min(keyValue, this.maximum);
          this.contractInfo.money = newVal;
      }else{
          this.contractInfo.money = keyValue
      }
    },

二、下拉框el-select在ipad上点击下拉框区域展开与隐藏无效

el-select下拉框在ipad上操作时输入框会获取到焦点，所以下拉框的展开与隐藏必须通过右侧icon图标进行操控，即使点击空白区域也无法控制下拉框，这样会给用户带来很不好的体验 可以通过事件visible-change（下拉框出现/隐藏时触发）控制下拉框失焦,详细代码如下

<el-select v-model="contractInfo.feetype" placeholder="请选择" ref='typeSel' @visible-change='handleTypeChange'>
	<el-option
		v-for="(item,index) in typeOptions"
		:key="index"
		:label="item.dicValue"
		:value="item.dicValue">
	</el-option>
</el-select>

三、ipad点击事件第一次无效，必须点击两次

table表格中for循环，操作栏需要有点击事件，在pc端可正常操作，但是ipad需要点击两次才能触发click事件

<el-table-column prop="statusText" label="操作" width="180px" style="text-align:left">
	<template slot-scope="scope" >
		<span  @click="handleAddCharge(scope.row)" class="textBtn" > 添加收费 </span>
	</template>
</el-table-column>

解决办法：在事件后面可以加debounce过滤器,最终代码如下

<el-table-column prop="statusText" label="操作" width="180px" style="text-align:left">
	<template slot-scope="scope" >
		<span  @click="handleAddCharge(scope.row)  | debounce" class="textBtn" > 添加收费 </span>
	</template>
</el-table-column>
